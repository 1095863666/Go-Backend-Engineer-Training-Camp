# 变量（Variable）——程序为什么需要变量？

------

> **一句话总结本章**

> **变量不是 Go 的概念，而是所有编程语言管理数据的基础。Go 对变量的设计，体现了它对安全性、可读性和工程性的追求。**

------

# 本章学习目标

完成本章后，你应该能够回答：

- 什么是变量？
- 为什么程序必须有变量？
- Go 为什么有三种声明方式？
- `var` 和 `:=` 到底有什么区别？
- Go 为什么不允许变量类型随意改变？
- 企业代码什么时候用 `var`，什么时候用 `:=`？

如果这些问题回答不了，这一章就还没有真正掌握。

------

# 第一节：如果没有变量，会发生什么？

先不要看 Go。

来看一个极端世界。

假设一门语言没有变量。

你想计算：

```
10 + 20
```

可以：

```
30
```

没问题。

但是：

如果今天。

用户输入：

```
18
```

明天：

输入：

```
25
```

后天：

输入：

```
99
```

程序怎么知道：

今天是多少？

答案：

**不知道。**

因为：

程序需要一个地方：

**保存数据。**

于是：

变量诞生了。

------

# 第二节：变量到底是什么？

很多书：

第一句话：

> 变量是存储数据的容器。

这句话：

**不能说错。**

但是：

不够准确。

真正理解应该是：

> **变量是程序对一块内存空间的一个名字。**

例如：

```
var age int = 18
```

很多初学者看到的是：

```
age = 18
```

实际上。

计算机更像这样：

```
          内存（Memory）

地址        内容

0x1000  ─────► 18
```

Go 编译器：

给：

```
0x1000
```

取了一个名字：

```
age
```

以后。

程序不用记：

```
0x1000
```

只需要：

```
age
```

------

# 第三节：为什么不用内存地址？

假设没有变量。

你每天写：

```
读取：

0x1000

修改：

0x2000

比较：

0x30A0
```

你觉得：

三天以后。

还能记住吗？

显然不能。

变量最大的价值：

不是存储。

而是：

> **让程序员不用直接操作内存地址。**

这是高级语言最大的贡献之一。

------

# 第四节：变量为什么叫 Variable？

英文：

Variable。

很多人翻译：

变量。

为什么？

因为：

它的值：

可以变化。

例如：

今天：

```
age = 18
```

明天：

```
age = 19
```

后天：

```
age = 20
```

名字：

没变。

值：

变了。

所以：

Variable。

------

与它对应的是：

Constant。

常量。

以后讲。

------

# 第五节：Go 为什么是静态类型语言？

这里开始进入 Go。

先看 Python。

```
age = 18

age = "Tom"

age = True
```

全部合法。

为什么？

Python：

动态类型。

变量：

可以不断改变类型。

------

Go：

```
var age int = 18

age = "Tom"
```

直接：

编译错误。

为什么？

很多初学者觉得：

Go 太严格。

实际上。

这是：

工程选择。

------

# 第六节：静态类型解决什么问题？

假设：

有一个项目：

500 万行代码。

一个函数：

```
UpdateUser(age)
```

如果：

`age`

可能：

是：

```
int

string

bool

float
```

阅读代码的人：

必须一路追踪：

这个变量：

到底是什么？

------

Go：

不会。

编译器：

已经知道：

```
age

↓

int
```

整个项目：

都不会变。

这就是：

静态类型。

------

# 第七节：Go 的第一种变量声明——var

终于开始写 Go。

最完整写法：

```
var age int = 18
```

拆开：

```
var

↓

声明变量
```

------

```
age

↓

变量名字
```

------

```
int

↓

变量类型
```

------

```
18

↓

初始值
```

整个意思：

> 创建一个名字叫 age 的变量，它保存一个 int，初始值为 18。

------

# 第八节：Go 为什么支持类型推导？

你可能觉得：

```
var age int = 18
```

里面：

`int`

是不是重复了？

因为：

18

本来就是整数。

Go 也这样认为。

于是：

允许：

```
var age = 18
```

编译器：

自动推导：

```
18

↓

int
```

注意：

这里：

不是没有类型。

而是：

**编译器替你写了。**

这一点非常重要。

------

# 第九节：为什么还有 := ?

Go 又提供：

```
age := 18
```

很多新人：

彻底懵了。

为什么：

已经有：

`var`

还要：

`:=`？

答案：

因为：

Go 非常重视：

**局部变量的开发效率。**

以后。

一个函数里面：

可能：

几十个变量。

如果全部：

```
var count int = 0

var userName string = "Tom"

var ok bool = true
```

太长。

于是：

Go 提供：

```
count := 0

userName := "Tom"

ok := true
```

短。

快。

容易读。

------

# 第十节：那么什么时候不能用 := ?

这是企业面试高频题。

例如：

包级变量：

```
package main

count := 0
```

错误。

为什么？

因为：

`:=`

只能：

用于：

**函数内部。**

包级：

必须：

```
var count = 0
```

------

为什么？

因为：

Go 希望：

全局变量：

更加明确。

不要：

隐藏声明。

------

# 第十一节：企业代码规范

这里开始讲真实项目。

例如：

Google 风格。

很多 Go 项目：

基本遵循：

------

包级变量：

```
var logger Logger
```

------

配置：

```
var config Config
```

------

函数内部：

```
user := User{}

err := LoadConfig()

count := 10
```

------

所以：

以后。

不要问：

哪个更高级。

没有。

只有：

适用场景。

------

# 第十二节：变量不是越多越好

很多新人：

喜欢：

```
a := 10

b := 20

c := a + b

d := c * 2

e := d + 1
```

实际上。

变量：

也是成本。

每增加一个变量。

阅读者：

就要：

记住一个名字。

所以：

好的 Go 代码：

变量：

通常：

名字清晰。

数量适中。

生命周期尽量短。

这是后面讲"变量作用域"时的重要思想。

------
