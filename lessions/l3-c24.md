# 第一节：为什么需要项目结构？

刚开始：

可能：

```
hello.go
```

里面：

```
package main

func main(){

}
```

------

可以。

------

但是项目增长：

比如：

流程引擎。

增加：

用户管理。

任务调度。

数据库。

日志。

API。

------

如果全部：

main.go：

会变成：

```
main.go

50000 lines
```

------

无法维护。

------

所以：

需要：

模块化。

------

# 第二节：什么是 Package？

Package：

包。

------

Go 代码组织单位。

------

例如：

目录：

```
user
 |
 ├── user.go
 └── password.go
```

------

里面：

```
package user
```

------

表示：

属于 user 包。

------

# 第三节：Package 的作用

主要三个：

------

## 1. 代码隔离

例如：

用户相关：

user package。

------

订单：

order package。

------

------

## 2. 复用

例如：

logger。

------

多个地方使用。

------

## 3. 控制访问权限

Go：

通过大小写。

------

# 第四节：Go 导出规则

Go 没有：

public/private。

------

使用：

大小写。

------

小写：

私有。

例如：

```
type user struct{

}
```

------

外部：

不能访问。

------

大写：

公开。

例如：

```
type User struct{

}
```

------

外部：

可以：

```
user.User{}
```

------

# 第五节：import 是什么？

例如：

```
import "fmt"
```

------

表示：

引入 package。

------

使用：

```
fmt.Println()
```

------

------

# 第六节：Go Module 出现以前的问题

以前：

Go 项目依赖：

GOPATH。

------

问题：

项目必须：

放固定目录。

------

依赖：

不好管理。

------

现代 Go：

使用：

# 第七节：什么是 Go Module？

Module：

一个 Go 项目。

------

核心文件：

```
go.mod
```

------

例如：

```
module github.com/company/flow-engine


go 1.23
```

------

表示：

项目名称。

------

# 第八节：创建 Module

进入项目：

```
go mod init github.com/myname/project
```

------

生成：

```
go.mod
```

------

例如：

```
flow-engine

└── go.mod
```

------

# 第九节：go.mod 内容

例如：

```
module flow-engine


go 1.23


require (

    github.com/gin-gonic/gin v1.10.0

)
```

------

解释：

module：

项目名字。

------

require：

依赖。

------

例如：

修改：

生成：

# 第十三节：什么是项目入口？

Go：

通常：

main package。

------

例如：

```
package main


func main(){

}
```

------

但是：

企业项目：

不会直接放根目录。

------

推荐：

cmd。

------

# 第十四节：cmd 目录

作用：

程序入口。

------

例如：

```
cmd

├── server

│   └── main.go


├── worker

│   └── main.go
```

------

为什么？

因为：

一个项目：

可能多个程序。

------

例如：

Flow Engine：

有：

API 服务。

Worker 服务。

Scheduler 服务。

------

# 第十五节：Flow Engine 结构

例如：

```
flow-engine

cmd

├── api

│   └── main.go


├── worker

│   └── main.go


└── scheduler

    └── main.go
```

------

三个程序：

共享代码。

------

# 第十六节：internal 目录

非常重要。

------

internal：

表示：

内部代码。

------

例如：

```
internal

├── flow

├── scheduler

└── repository
```

------

外部项目：

不能 import。

------

# 第十七节：为什么需要 internal？

保护架构。

------

例如：

你的：

数据库实现。

------

不希望：

别人：

直接调用。

------

所以：

放：

internal。

------

# 第十八节：pkg 目录

pkg：

公共库。

------

例如：

别人也可以使用。

------

例如：

```
pkg

├── logger

├── httpclient

└── validator
```

------

------

# 第十九节：internal 和 pkg 区别

重点：

| 目录     | 用途           |
| -------- | -------------- |
| internal | 项目内部       |
| pkg      | 可复用公共代码 |

------

例如：

Flow Engine：

数据库：

internal。

------

日志：

pkg。

------

# 第二十三节：main.go 应该做什么？

很多新人：

把业务写 main。

错误。

------

main：

只负责：

组装。

------

例如：

```
func main(){

    db:=InitDB()

    service:=NewService(db)

    server:=NewServer(service)

    server.Run()

}
```

------

叫：

依赖注入。

------

# 第二十四节：不要这样写

错误：

```
func main(){

    createUser()

    updateDB()

    sendEmail()

}
```

------

问题：

无法测试。

无法复用。

------

# 第二十五节：Go 项目初始化实战

创建：

cd flow-engine

------

```
go mod init flow-engine
```

```
cmd/server/main.go
```

------

    )

go run ./cmd/server
