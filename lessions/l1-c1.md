# 第一章

## 软件工程的发展

在正式讲 Go 之前。

先回答一个问题。

> **为什么每隔十几年，就会出现一门新的编程语言？**

例如：

```
1972

C
```

然后。

```
1983

C++
```

后来。

```
1995

Java
```

同一年。

```
Python
```

再后来。

```
2009

Go
```

很多初学者看到这里。

第一个问题就是：

> 已经有这么多语言了，为什么还要发明 Go？

这是一个非常好的问题。

答案只有一句话：

> **每一种语言，都是为了解决上一代语言解决不好的问题。**

请记住这句话。

以后你学习 Rust。

学习 Kotlin。

学习 Swift。

都会发现同样的规律。

------

## C 解决了什么问题？

在 C 出现之前。

很多程序都是汇编写的。

例如：

```
MOV AX,10

ADD BX,20

JMP ...
```

写程序非常痛苦。

不同 CPU。

代码几乎不能复用。

于是。

Dennis Ritchie 发明了 C。

第一次让程序员能够：

> 用接近自然语言的方式写操作系统。

Unix。

就是 C 最伟大的作品之一。

------

## C++ 为什么出现？

随着软件越来越复杂。

程序越来越大。

大家开始发现：

只有函数已经不够用了。

于是。

Bjarne Stroustrup 发明了：

C++。

加入：

- 类（Class）
- 继承（Inheritance）
- 多态（Polymorphism）
- 模板（Template）

那个年代。

面向对象成为主流。

很多人认为：

> 软件复杂，就需要更强大的抽象能力。

于是：

C++ 成为了大型软件的重要选择。

------

## Java 为什么成功？

到了 90 年代。

互联网开始出现。

Sun 提出了一个非常著名的口号。

> Write Once, Run Anywhere.

Java 最大的贡献。

不是语法。

而是：

**JVM。**

它解决了：

跨平台。

自动垃圾回收。

丰富的标准库。

企业级开发。

那个时代。

Java 几乎统治了企业软件。

------

## Python 为什么流行？

Python 出现以后。

它没有追求最快。

它追求：

> **开发效率。**

例如：

下面两段代码。

Python：

```
for i in range(10):
    print(i)
```

C++：

```
for(int i=0;i<10;i++){
    std::cout<<i<<std::endl;
}
```

Python 更短。

更容易读。

于是。

越来越多：

自动化。

运维。

AI。

数据分析。

开始使用 Python。

------

到这里。

世界已经拥有：

最快的：

C++

企业：

Java

开发效率：

Python

那么。

Go 为什么还会出现？

## 📖 本章小结

今天你只需要记住三句话：

1. **新语言不是为了替代旧语言，而是为了解决新的工程问题。**
2. **Go 的出现不是偶然，而是互联网时代发展的结果。**
3. **真正理解一门语言，要先理解它诞生时想解决什么问题。**
