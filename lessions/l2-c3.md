# 常量（Constant）——为什么程序需要永远不会改变的数据？

------

> **一句话总结本章**

> **变量描述程序运行中的变化，而常量描述程序世界中不变的规则。**

------

# 本章学习目标

完成本章后，你应该能够回答：

- 什么是常量？
- 为什么不能全部使用变量？
- Go 为什么设计 `const`？
- 常量和变量最大的区别是什么？
- Go 的常量为什么比很多语言更灵活？
- 企业开发中哪些数据应该使用常量？

------

# 第一节：如果只有变量，会发生什么？

继续上一章。

我们已经知道：

变量：

```
age := 18
```

可以改变。

例如：

```
age = 19
```

没有问题。

但是。

有一种数据。

天生不应该改变。

例如：

圆周率。

```
3.1415926...
```

今天：

不会变。

明天：

不会变。

十年以后：

还是不会变。

------

再例如：

一年：

```
12 个月
```

不会因为程序运行：

变成：

13。

------

再例如：

HTTP：

成功：

```
200
```

服务器错误：

```
500
```

这些都是：

规则。

不是状态。

------

于是。

程序需要一种能力：

告诉编译器：

> **这个值，从出生开始，就不会改变。**

于是。

出现了：

Constant。

------

# 第二节：什么是常量？

很多书：

写：

> 常量是不能修改的变量。

我认为：

这是一个错误理解。

------

为什么？

因为：

常量：

**不是变量。**

变量：

强调：

变化。

------

常量：

强调：

规则。

------

举一个现实例子。

你的年龄：

今天：

18。

明年：

19。

属于：

变量。

------

你的身份证号码。

出生以后。

基本：

不会改变。

更接近：

常量。

------

程序也是一样。

例如：

```
const MaxRetry = 3
```

这里。

不是因为：

3

不能修改。

而是：

业务规则：

最多：

重试：

3 次。

------

重点：

**业务规则。**

------

# 第三节：Go 为什么需要 const？

来看一个真实 Bug。

假设：

程序里面：

到处写：

```
retry > 3
```

后来：

产品经理：

要求：

改成：

5。

于是：

程序员开始：

全项目搜索：

```
3
```

问题来了。

你能确定：

所有：

3

都是：

重试次数吗？

例如：

还有：

```
age > 3

month == 3

version == 3
```

完全：

混在一起。

------

如果：

一不小心：

改错。

Bug：

产生。

------

于是。

正确写法：

```
const MaxRetry = 3
```

以后：

修改：

```
const MaxRetry = 5
```

整个项目：

自动更新。

------

这就是：

**消除 Magic Number（魔法数字）。**

------

# 第四节：什么是 Magic Number？

企业开发：

非常讨厌：

这种代码：

```
if retry > 3 {

}
```

请问：

3

什么意思？

没人知道。

------

更好的：

```
if retry > MaxRetry {

}
```

现在：

不用注释。

大家都知道：

这是：

最大重试次数。

------

优秀代码：

应该：

自己会说话。

------

# 第五节：Go 的 const

正式开始。

最简单：

```
const MaxRetry = 3
```

表示：

创建：

一个：

永远不会改变的常量。

------

多个：

```
const (

    MaxRetry = 3

    MaxWorker = 10

)
```

企业：

通常：

这样写。

------

# 第六节：const 为什么不能修改？

例如：

```
const PI = 3.14

PI = 5
```

编译：

直接失败。

为什么？

因为：

编译器：

已经认为：

它永远：

不会改变。

------

这里。

有一个重要思想。

变量：

运行时：

存在。

------

很多常量：

编译时：

已经确定。

------

所以：

Go 编译器：

甚至：

可以：

直接：

替换。

例如：

```
const A = 10

x := A + 5
```

编译器：

可能：

直接变成：

```
x := 15
```

这叫：

**常量折叠（Constant Folding）**。

以后讲编译器：

还会详细分析。

------

# 第七节：Go 常量比很多语言更强

这里。

Go 很特别。

例如：

```
const A = 10
```

注意。

这里：

没有：

```
int
```

为什么？

因为：

Go 的常量：

默认：

是：

**无类型常量（Untyped Constant）。**

------

什么意思？

例如：

```
const A = 10

var x int = A

var y int64 = A

var z float64 = A
```

全部：

合法。

------

为什么？

因为：

A

不是：

一个真正的：

int。

------

而是：

一个：

编译期常量。

------

这是 Go 一个非常优雅的设计。

后面讲类型系统：

我们再深入。

------

# 第八节：什么时候不用 const？

不是所有固定值：

都应该：

const。

例如：

配置文件：

```
retry: 5
```

为什么？

因为：

用户：

可能：

修改。

------

数据库地址：

```
mysql:3306
```

应该：

配置。

不是：

const。

------

所以：

判断标准：

不是：

今天有没有修改。

而是：

**设计上，它是否应该改变。**

------

# 第九节：企业实践

来看几个真实例子。

------

HTTP：

```
const StatusOK = 200
```

------

权限：

```
const RoleAdmin = "admin"
```

------

重试：

```
const MaxRetry = 3
```

------

默认分页：

```
const DefaultPageSize = 20
```

------

但是：

数据库密码：

×

```
const DBPassword = "123456"
```

绝对不要。

应该：

配置文件。

环境变量。

或者：

密钥管理系统。

------

# 第十节：Go 的 iota（预告）

还有一个：

Go 独有：

设计。

例如：

```
const (

    Red = iota

    Green

    Blue

)
```

以后。

自动：

```
Red = 0

Green = 1

Blue = 2
```

这一章。

先知道：

存在。

下一章。

专门讲：

iota。

因为：

它背后还有很多设计思想。
