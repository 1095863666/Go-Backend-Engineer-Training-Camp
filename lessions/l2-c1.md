# 第四节：理解 Go 环境

执行：

```
go env
```

你会看到很多配置。

例如：

```
GOROOT

GOPATH

GOMODCACHE

GOOS

GOARCH
```

------

现在不用全部记。

我们先理解三个。

------

# 1. GOROOT

是什么？

Go 安装目录。

例如：

```
/usr/local/go
```

里面：

包含：

Go 自己的东西。

例如：

```
src/

pkg/

bin/
```

------

注意：

你一般：

不要修改它。

------

# 2. GOPATH

早期 Go：

代码存放位置。

以前：

所有项目：

必须放：

```
$GOPATH/src
```

------

但是：

现在：

Go Module 出现以后。

基本不用关心。

------

# 3. GOMODCACHE

依赖缓存位置。

例如：

你使用：

Gin。

Go 下载：

保存。

以后：

重复使用。

# 第五节：第一个 Go 程序

创建目录：

```
mkdir hello-go

cd hello-go
```

------

创建：

```
main.go
```

------

写入：

```
package main

import "fmt"

func main() {

    fmt.Println("Hello Go")

}
```

------

运行：

```
go run main.go
```

输出：

```
Hello Go
```

------

恭喜。

你的第一个 Go 程序运行成功。

------

但是。

我们不要急着结束。

现在分析：

每一行是什么意思。

------

# 第六节：package main

第一行：

```
package main
```

是什么意思？

------

Go 程序由：

package

组成。

------

package：

类似：

一个代码模块。

------

例如：

```
user

order

payment
```

------

但是：

可执行程序：

必须有：

```
package main
```

------

为什么？

因为：

Go 编译器需要知道：

入口在哪里。

------

# 第七节：func main()

代码：

```
func main(){

}
```

这是：

程序入口。

------

类似：

C：

```
int main()
```

Java：

```
public static void main()
```

------

程序启动：

从：

main：

开始执行。

------

# 第八节：import

代码：

```
import "fmt"
```

表示：

引入包。

------

fmt：

format。

负责：

格式化输入输出。

------

例如：

```
fmt.Println()
```

------

以后：

你会大量使用：

标准库。

例如：

```
net/http

database/sql

context

sync
```

------

# 第九节：Go 命令体系

现在进入重点。

Go 工具链。

------

## 1. go run

作用：

运行程序。

------

例如：

```
go run main.go
```

流程：

```
代码

↓

临时编译

↓

执行

↓

删除临时文件
```

------

适合：

开发测试。

------

------

## 2. go build

作用：

编译。

------

执行：

```
go build
```

生成：

```
hello-go.exe
```

或者：

Linux：

```
hello-go
```

------

区别：

go run：

运行。

go build：

生成程序。

------

企业部署：

通常：

使用 build。

------

# 第十节：Go 跨平台编译

这是 Go 很强的地方。

例如：

你现在：

Windows。

想编译：

Linux 程序。

------

设置：

```
set GOOS=linux

set GOARCH=amd64
```

然后：

```
go build
```

------

生成：

Linux 可执行文件。

------

为什么重要？

服务器：

通常：

Linux。

开发：

可能：

Windows。

------

Go：

天然支持。

------

# 第十一节：go fmt

这是 Go 最重要命令之一。

------

执行：

```
go fmt
```

自动格式化。

------

例如：

你写：

```
func main(){
fmt.Println("hello")
}
```

------

执行：

```
go fmt
```

变成：

```
func main() {
    fmt.Println("hello")
}
```

------

为什么重要？

因为：

团队不需要讨论格式。

------

Go 规定：

只有一种标准格式。

------

# 第十二节：go test

Go 内置测试。

------

例如：

文件：

```
add.go

add_test.go
```

------

测试：

```
go test
```

------

输出：

```
PASS
```

------

以后企业项目：

测试非常重要。

------

# 第十三节：go mod

这是现代 Go 项目的核心。

------

创建：

```
go mod init example.com/hello
```

生成：

```
go.mod
```

------

内容：

类似：

```
module example.com/hello

go 1.24
```

------

它表示：

这个项目：

是一个 Go Module。

------

以前：

Go 不知道：

项目依赖。

------

现在：

go.mod：

记录：

- 项目名称
- Go 版本
- 第三方依赖

------

# 第十四节：标准 Go 项目结构

现在创建：

企业项目结构。

------

推荐：

```
hello-go

├── go.mod

├── main.go

├── README.md

├── cmd/

├── internal/

├── pkg/
```

------

现在不用全部理解。

先认识。

------

## cmd

程序入口。

例如：

```
cmd/server/main.go
```

------

## internal

内部业务代码。

别人不能引用。

------

## pkg

公共代码。

------

以后：

企业项目：

基本都会看到。
