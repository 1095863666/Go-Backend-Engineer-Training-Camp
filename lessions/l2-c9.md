# 指针（Pointer）——为什么 Go 保留指针，却禁止指针运算？

------

> **一句话总结本章**

> **指针保存的是内存地址，它让程序可以直接操作数据；Go 保留指针用于性能和表达，但限制指针运算保证安全。**

------

# 本章学习目标

完成本章后，你应该能够：

- 理解什么是内存地址
- 理解指针是什么
- 理解 `&` 和 `*`
- 理解值传递和指针传递
- 理解 Go 为什么需要指针
- 理解 Go 为什么删除指针运算
- 知道什么时候应该使用指针

------

# 第一节：为什么需要指针？

先回顾变量。

例如：

```
age := 18
```

计算机内部：

类似：

```
内存

地址        数据

0x1000      18
```

变量：

```
age
```

只是：

这个地址的名字。

------

现在：

有一个函数：

```
func change(age int){

    age = 20

}
```

调用：

```
age := 18

change(age)

fmt.Println(age)
```

结果：

```
18
```

为什么？

因为：

传进去的是：

数据复制。

------

# 第二节：什么叫值传递？

Go 默认：

全部参数传递：

都是：

值传递。

例如：

```
func test(x int){

}
```

调用：

```
a := 10

test(a)
```

内存：

开始：

```
a

↓

10
```

调用：

复制：

```
a

↓

10


x

↓

10
```

两个：

完全不同。

------

所以：

修改：

x。

不会影响：

a。

------

# 第三节：如果想修改原数据怎么办？

我们需要：

告诉函数：

不要复制数据。

直接找到：

原来的位置。

怎么办？

给地址。

------

于是：

出现：

指针。

------

# 第四节：什么是指针？

指针：

不是数据。

它保存：

另一个变量的地址。

例如：

```
age := 18

p := &age
```

这里：

`&`

表示：

取地址。

------

内存：

可能：

```
地址        内容


0x1000      18

0x2000      0x1000
```

变量：

```
age

↓

0x1000
```

指针：

```
p

↓

0x1000
```

------

所以：

p 保存的是：

age 的地址。

------

# 第五节：& 是什么意思？

代码：

```
p := &age
```

读：

> 创建一个指针 p，它保存变量 age 的地址。

------

例如：

```
name := "Tom"

ptr := &name
```

ptr：

保存：

name：

在哪里。

------

# 第六节：* 是什么意思？

有两个地方：

容易混。

------

## 第一种：声明指针

例如：

```
var p *int
```

意思：

p 是：

指向 int 的指针。

------

## 第二种：解引用

例如：

```
*p
```

意思：

访问：

p 指向的数据。

------

例如：

```
age := 18

p := &age

*p = 20
```

发生：

```
之前：

age

↓

18


之后：

age

↓

20
```

------

# 第七节：完整例子

看：

```
package main

import "fmt"

func main(){

    age := 18

    p := &age

    *p = 30

    fmt.Println(age)

}
```

输出：

```
30
```

为什么？

因为：

修改：

不是：

p。

而是：

p 指向的：

age。

------

# 第八节：为什么 Go 需要指针？

有人问：

Go 不是有 GC 吗？

为什么还需要指针？

原因：

三个。

------

## 原因 1：修改原对象

例如：

函数：

修改：

结构体。

------

## 原因 2：避免复制大对象

例如：

```
type User struct {

    Name string

    Age int

}
```

如果：

传递：

100MB 数据。

复制：

浪费。

------

## 原因 3：表达不存在

例如：

数据库查询。

用户：

可能不存在。

怎么办？

```
*User
```

可以：

```
nil
```

表示：

没有。

------

# 第九节：结构体为什么大量使用指针？

例如：

用户：

```
type User struct {

    Name string

    Age int

}
```

函数：

```
func UpdateUser(user User){

}
```

复制。

------

改：

```
func UpdateUser(user *User){

}
```

传地址。

------

企业代码：

大量：

```
func SaveUser(user *User)
```

------

# 第十节：Go 为什么禁止指针运算？

C：

可以：

```
ptr++
```

意思：

地址移动。

------

例如：

数组：

```
1000

1004

1008

1012
```

ptr++：

移动。

------

Go：

不允许。

为什么？

因为：

危险。

------

例如：

你移动错：

访问：

不存在区域。

可能：

崩溃。

------

Go 的设计：

是：

让程序员拥有：

必要控制。

但不要：

无限自由。

------

# 第十一节：Go 指针 vs C 指针

|          | C    | Go   |
| -------- | ---- | ---- |
| 保存地址 | ✔    | ✔    |
| 修改数据 | ✔    | ✔    |
| 指针运算 | ✔    | ❌    |
| 手动释放 | ✔    | ❌    |
| GC       | ❌    | ✔    |

------

Go：

保留：

80% 好处。

去掉：

80% 风险。

------

# 第十二节：nil 指针

重要。

例如：

```
var p *int
```

现在：

p：

没有地址。

值：

```
nil
```

------

执行：

```
*p = 10
```

错误。

因为：

没有目标。

------

类似：

你拿了一张：

空地址纸条。

但是：

没有房子。

------

# 第十三节：企业实践

什么时候使用指针？

------

## 使用指针：

### 1

结构体比较大。

例如：

```
func Process(ctx *Context)
```

------

### 2

需要修改对象。

例如：

```
func Update(user *User)
```

------

### 3

需要表示不存在。

例如：

```
func FindUser(id int) *User
```

------

## 不使用指针：

小数据：

```
func Add(a int,b int)
```

没有必要。

------

# 第十四节：Flow Engine 实战

我们的流程：

```
type Node struct {

    ID NodeID

    Status NodeStatus

}
```

执行：

```
func ExecuteNode(node *Node){

    node.Status = NodeSuccess

}
```

为什么？

因为：

执行节点：

需要改变：

节点状态。

------

如果：

写：

```
func ExecuteNode(node Node)
```

修改：

只是：

副本。

外面的：

Node：

不会变化。

------

# 第十五节：指针和 Slice 的关系

这里提前埋一个坑。

为什么：

Slice：

传函数：

不用：

`*[]int`？

例如：

```
func Add(nums []int)
```

可以。

因为：

Slice 本身：

已经包含：

指针。

------

Slice：

本质：

```
ptr

len

cap
```

复制：

只是复制：

这三个字段。

------

但是：

如果 append：

可能：

改变指向。

所以：

需要：

返回。

------

这也是：

为什么 Slice 和 Pointer 经常一起考。
