# Slice（二）——len、cap 与 append 的底层原理

------

> **一句话总结本章**

> **Slice 本身不会存储数据，它只是管理数组的一层描述，而 append 的本质就是判断：继续使用原数组，还是申请一块新的数组。**

- append 为什么必须接收返回值？

- append 什么时候会扩容？

# 第一节：先回顾 Slice

上一章：

我们知道：

```
arr := [5]int{10, 20, 30, 40, 50}

s := arr[1:4]
```

很多人看到的是：

```
20
30
40
```

真正应该看到的是：

```
数组

+----+----+----+----+----+
| 10 | 20 | 30 | 40 | 50 |
+----+----+----+----+----+
       ^
       |
       s
```

Slice：

没有复制。

只是：

指向数组。

------

# 第二节：什么是 len？

来看：

```
arr := [5]int{10,20,30,40,50}

s := arr[1:4]
```

Slice：

包含：

```
20

30

40
```

所以：

```
len(s)
```

结果：

```
3
```

为什么？

因为：

当前：

可以访问：

三个元素。

所以：

> **len = 当前能够访问的数据数量。**

------

# 第三节：什么是 cap？

这是 Go 初学者最困惑的地方。

还是：

```
arr := [5]int{10,20,30,40,50}

s := arr[1:4]
```

来看图。

```
数组

索引：

0   1   2   3   4

10 20 30 40 50
   ↑
   s
```

Slice：

从：

索引：

1

开始。

一直：

到数组结束。

还能看到：

```
20

30

40

50
```

所以：

```
cap(s)
```

结果：

```
4
```

很多人问：

为什么不是：

3？

因为：

cap：

不是：

当前长度。

而是：

> **从 Slice 起点，到底层数组结尾，还剩多少空间。**

------

# 第四节：为什么需要 cap？

假设：

没有 cap。

程序：

执行：

```

```

编译器：

怎么知道：

后面：

还有没有空间？

不知道。

于是：

每次：

append：

都只能：

重新申请：

数组。

效率：

非常低。

------

所以：

Slice：

提前记录：

```
还能放几个元素。
```

这就是：

cap。

------

# 第五节：append 做了什么？

来看：

```

s := arr[1:3]
```

注意：

现在：

Slice：

只有：

```
20

30
```

但是：

是多少？

```
4
```

因为：

后面：

还有：

```
40

50
```

空间。

------

执行：

```

```

真正发生：

```
原数组

10

20

30

99

50
```

注意。

40：

没有了。

因为：

append：

直接：

写到：

原数组。

------

# 第六节：为什么会修改原数组？

因为：

没有扩容。

还是：

同一个数组。

例如：

```
数组：

10

20

30

40

50

↓

append

↓

10

20

30

99

50
```

没有：

复制。

没有：

新数组。

------

所以：

原数组：

变了。

# 第七节：什么时候会扩容？

来看：

```
arr := [3]int{1,2,3}

s := arr[:]
```

现在：

```
len = 3

cap = 3
```

已经：

满了。

------

执行：

```
s = append(s,4)
```

还能放吗？

不能。

因为：

cap：

已经：

用完。

------

Go：

只能：

申请：

新数组。

例如：

```
旧数组

1

2

3

↓

新数组

1

2

3

4
```

然后：

Slice：

指向：

新数组。

------

原数组：

不变。

------

# 第八节：为什么 append 必须赋值？

很多新人：

写：

```

```

然后：

奇怪：

为什么：

没变化？

------

因为：

append：

返回：

新的 Slice。

正确：

```

```

为什么？

因为：

可能：

发生：

扩容。

------

如果：

扩容。

地址：

改变。

旧 Slice：

已经：

失效。

所以：

必须：

接收：

返回值。

------

# 第九节：append 的两种情况

以后。

脑子里：

一定要：

形成：

这张图。

------

## 第一种：

还有容量。

```

↓

```

------

## 第二种：

容量不足。

```

↓

申请新数组

↓

```

这就是：

append。

全部原理。

    nums = append(nums,100)

认为：

```

2

其实：

为什么？

因为：

已经：

外面：

还是：

旧数组。

------

所以：

企业：

    nums = append(nums,100)

```

```

```

这是：

Go：

经典写法。
