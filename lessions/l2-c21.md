# Select 高级并发控制——构建生产级任务调度器

------

> **一句话总结本章**

> **select 负责监听多个并发事件，context 负责控制任务生命周期，两者结合构成 Go 服务中的取消、超时和优雅退出机制。**

------

- 理解 nil channel

- 设计 Flow Engine Worker 生命周期

# 第一节：为什么需要 select？

上一章：

我们写：

```
result := <-ch
```

------

问题：

只能等待：

一个事件。

------

但是现实：

通常：

等待多个情况。

例如：

任务执行：

可能：

1. 正常完成
2. 超时
3. 用户取消
4. 服务关闭

------

所以：

需要：

同时监听。

------

# 第二节：select 基础

例如：

```
select {

case msg := <-ch1:

    fmt.Println(msg)


case msg := <-ch2:

    fmt.Println(msg)

}
```

------

含义：

等待：

ch1 或 ch2。

------

谁先有数据：

执行谁。

------

# 第三节：select 的执行规则

例如：

```
select {

case <-ch1:

    fmt.Println("one")


case <-ch2:

    fmt.Println("two")

}
```

------

如果：

两个同时准备好了。

------

Go：

随机选择。

------

为什么？

避免：

某个 Channel 永远饥饿。

------

# 第四节：select 没有 default 会怎样？

例如：

```
select {

case msg := <-ch:

}
```

------

如果：

没有数据。

------

当前 Goroutine：

阻塞。

------

等待。

------

# 第五节：select + default

例如：

```
select {

case msg := <-ch:

    fmt.Println(msg)


default:

    fmt.Println("no data")

}
```

------

特点：

立即返回。

------

叫：

非阻塞 select。

------

# 第六节：什么时候使用 default？

例如：

检查：

任务队列。

------

```
select {

case task := <-queue:

    execute(task)


default:

    idle()

}
```

------

但是：

不要滥用。

------

因为：

可能：

CPU 空转。

------

# 第七节：超时控制

这是最常用。

------

例如：

调用外部服务。

最多：

3 秒。

------

代码：

```
select {

case result := <-resultCh:

    return result


case <-time.After(
    3*time.Second,
):

    return timeout

}
```

------

流程：

```
开始等待

 |

 +-----------+

 |           |

结果       3秒

 |           |

返回       超时
```

------

# 第八节：为什么不能只 sleep？

错误：

```
time.Sleep(time.Second*3)
```

------

因为：

你不知道：

什么时候完成。

------

例如：

任务：

1 秒完成。

你仍然等待：

3 秒。

------

select：

事件驱动。

------

# 第九节：Channel 和超时结合

真实例子：

```
func request() error {

    result := make(chan error)


    go func(){

        err:=callAPI()

        result <- err

    }()


    select {

    case err:=<-result:

        return err


    case <-time.After(time.Second*5):

        return errors.New("timeout")

    }

}
```

------

------

# 第十节：问题：超时后 Goroutine 怎么办？

这是高级问题。

刚才：

如果：

5 秒超时。

但是：

callAPI：

继续执行。

------

结果：

Goroutine 泄漏。

------

怎么办？

需要：

取消机制。

# 第十一节：Context 是什么？

context：

上下文。

------

作用：

传递：

- 取消信号
- 超时
- 请求信息

------

定义：

```
context.Context
```

------

Go 后端：

几乎所有服务都会使用。

# 第十二节：最简单 Context

创建：

```
ctx := context.Background()
```

------

这是：

根 Context。

------

通常：

不会直接用。

------

# 第十三节：WithCancel

创建：

可取消 Context。

------

代码：

```
ctx,cancel :=
    context.WithCancel(
        context.Background(),
    )


defer cancel()
```

------

产生：

两个东西：

ctx

cancel函数

------

调用：

```
cancel()
```

------

通知：

所有监听者。

------

# 第十四节：如何监听取消？

使用：

```
<-ctx.Done()
```

------

例如：

```
func worker(ctx context.Context){

    for {

        select {

        case <-ctx.Done():

            return


        default:

            work()

        }

    }

}
```

------

------

# 第十五节：Context 工作模型

结构：

```
Parent Context

        |

        |

     Child Context

        |

        |

   Goroutine
```

------

取消：

从上往下传播。

------

# 第十六节：WithTimeout

非常常用。

------

例如：

5 秒超时。

------

```
ctx,cancel :=
context.WithTimeout(
    context.Background(),
    time.Second*5,
)

defer cancel()
```

------

5 秒后：

自动：

cancel。

------

------

# 第十七节：WithDeadline

区别：

Timeout：

多久以后。

------

Deadline：

指定时间。

------

例如：

```
deadline :=
time.Now().Add(time.Hour)


ctx,cancel :=
context.WithDeadline(
    context.Background(),
    deadline,
)
```

------

------

request context。

# 第十九节：Flow Engine 使用 Context

非常重要。

------

节点：

接口：

```
type NodeExecutor interface {

    Execute(
        ctx context.Context,
    ) error

}
```

------

为什么？

因为：

节点可能：

运行很久。

------

需要：

取消。

------

例如：

```
func (n *SQLNode) Execute(
ctx context.Context,
) error {


select {


case <-ctx.Done():

    return ctx.Err()


case result:=<-dbResult:

    save(result)

}


}
```

------

# 第二十节：Worker 优雅退出

生产服务：

不能：

直接结束。

------

例如：

服务器升级。

------

流程：

```
收到 SIGTERM

↓

停止接收新任务

↓

等待旧任务完成

↓

关闭 Worker

↓

退出
```

------

叫：

Graceful Shutdown。

------

# 第二十一节：Worker 生命周期设计

例如：

```
func worker(ctx context.Context){

for {


select {


case <-ctx.Done():

    return


case task:=<-tasks:

    execute(task)

}


}

}
```

------

当：

ctx cancel。

------

worker：

自动退出。

------

```
 Worker1  Worker2  Worker3

    |         |         |

    +---------+---------+

              |

       Result Channel

              |

        Flow State
```

------

控制：

Context。

# 第二十三节：nil Channel 特性

特殊。

------

如果：

```
var ch chan int
```

默认：

nil。

------

发送：

```
ch <- 1
```

永久阻塞。

------

接收：

```
<-ch
```

永久阻塞。

------

close：

panic。

# 第二十四节：nil Channel 的高级用途

动态关闭 select 分支。

例如：

```
var timer <-chan time.Time


select {

case <-timer:

}
```

------

nil：

这个 case 永远不会执行。

------

用于：

动态控制流程。
