# Go 类型系统（Type System）——为什么 Go 如此强调类型？

------

> **一句话总结本章**

> **类型不是限制程序员，而是帮助程序员提前发现错误，并让代码表达真实业务含义。**

------

# 本章学习目标

完成本章后，你应该能够理解：

- 什么是类型？
- 为什么计算机需要类型？
- 静态类型有什么价值？
- Go 的基本类型有哪些？
- 类型定义和类型别名有什么区别？
- 为什么 `type UserID int` 不等于 `int`？
- 企业代码为什么重视类型设计？

------

# 第一节：如果没有类型，会发生什么？

我们先想象：

一个没有类型的世界。

例如：

```
10
```

计算机看到：

10。

但是：

它不知道：

这是：

年龄？

价格？

数量？

还是：

用户 ID？

------

例如：

```
price := 100
age := 100
```

两个变量：

数值一样。

但是：

业务意义完全不同。

------

如果程序允许：

```
price + age
```

语法正确。

但是：

业务错误。

------

所以：

类型第一个作用：

不是告诉计算机：

"这是数字。"

而是：

告诉程序：

> **这个数字代表什么。**

------

# 第二节：类型到底是什么？

简单理解：

类型包含两个信息。

------

## 第一：数据格式

例如：

整数：

```
4 bytes
```

字符串：

```
一组字符
```

------

## 第二：允许的操作

例如：

整数：

可以：

```
加减乘除
```

------

字符串：

可以：

```
拼接
```

------

例如：

```
age + 10
```

合理。

------

但是：

```
age + "Tom"
```

没有意义。

------

类型：

帮助编译器：

阻止这种错误。

------

# 第三节：动态类型 vs 静态类型

这是面试高频。

------

## Python

动态类型：

```
x = 10

x = "hello"
```

运行期间：

才知道：

x 是什么。

------

## Go

静态类型：

```
var x int

x = "hello"
```

编译失败。

------

区别：

不是：

谁高级。

而是：

设计目标不同。

------

# 第四节：动态类型的优点

动态语言：

优势：

开发快。

例如：

Python：

```
user.name
```

不用声明：

类型。

------

适合：

- 脚本
- 数据分析
- 快速验证

------

# 第五节：静态类型的优势

Go：

优势：

大型工程。

------

例如：

500 人团队。

代码：

100 万行。

------

如果没有类型：

很多错误：

直到运行：

才发现。

------

Go：

编译阶段：

提前发现。

------

例如：

```
func SendEmail(userID int){

}
```

调用：

```
SendEmail("100")
```

直接：

编译失败。

------

错误：

在上线前：

消灭。

------

# 第六节：Go 的基本类型

Go 内置类型：

主要：

下面几类。

------

# 1. 整数类型

## int

最常用：

```
var age int = 18
```

------

## int8

8 位：

范围：

```
-128 ~ 127
```

------

## int16

------

## int32

------

## int64

------

无符号：

```
uint
uint8
uint64
```

------

企业建议：

普通业务：

使用：

```
int
```

即可。

------

特殊场景：

例如：

协议。

文件格式。

才使用：

固定大小。

------

# 2. 浮点类型

```
float32

float64
```

例如：

```
var price float64 = 99.9
```

------

金融系统：

通常：

不直接使用 float。

为什么？

因为：

精度问题。

例如：

0.1 + 0.2

可能：

不是：

0.3。

------

金融：

通常：

使用：

整数分。

例如：

```
99.99 元

=

9999 分
```

------

# 3. bool

只有：

两个值：

```
true

false
```

例如：

```
running := true
```

------

流程引擎：

大量使用。

例如：

```
if node.Enabled {

}
```

------

# 4. string

字符串。

例如：

```
name := "Tom"
```

------

Go 的 string：

本质：

是：

字节序列。

不是：

字符数组。

后面：

讲 Unicode 时：

深入。

------

# 第七节：Go 的零值（Zero Value）

这是 Go 非常重要的设计。

------

很多语言：

变量必须初始化。

例如：

Java：

```
int age;
```

可能：

报错。

------

Go：

允许：

```
var age int
```

默认：

```
0
```

------

不同类型：

默认值：

| 类型    | 零值  |
| ------- | ----- |
| int     | 0     |
| float   | 0     |
| bool    | false |
| string  | ""    |
| pointer | nil   |

------

为什么？

因为：

Go 希望：

变量永远有一个合理状态。

------

# 第八节：为什么零值很重要？

看结构体。

例如：

```
type User struct {

    Name string

    Age int

}
```

创建：

```
user := User{}
```

自动：

得到：

```
Name = ""

Age = 0
```

------

不用：

写：

```
User{
    Name:"",
    Age:0,
}
```

------

这让：

Go 代码：

非常简洁。

------

# 第九节：类型定义（Type Definition）

现在进入重点。

------

代码：

```
type UserID int
```

是什么意思？

------

很多新人认为：

```
UserID = int
```

错误。

------

正确：

创建：

一个新的类型。

------

例如：

```
type UserID int

var id UserID = 100
```

------

它底层：

像 int。

但是：

它不是 int。

------

# 第十节：为什么需要新类型？

看例子。

------

错误：

```
func GetUser(id int){

}

func DeleteUser(id int){

}
```

调用：

```
GetUser(orderID)
```

如果：

orderID：

也是：

int。

编译器：

不知道错了。

------

但是：

设计类型：

```
type UserID int

type OrderID int
```

------

然后：

```
func GetUser(id UserID)

func DeleteOrder(id OrderID)
```

------

传错：

编译失败。

------

这就是：

类型安全。

------

# 第十一节：类型就是业务表达

优秀代码：

不是：

```
func Pay(id int)
```

而是：

```
func Pay(orderID OrderID)
```

------

不是：

```
func Send(id int)
```

而是：

```
func SendEmail(userID UserID)
```

------

类型：

就是：

代码注释。

------

# 第十二节：类型别名 alias

Go 还有：

另一种：

```
type UserID = int
```

注意：

有一个：

=

------

区别：

------

## 类型定义

```
type UserID int
```

创建新类型。

------

## 类型别名

```
type UserID = int
```

只是另一个名字。

------

例如：

```
var a UserID

var b int

a = b
```

------

如果：

类型定义：

错误。

------

如果：

别名：

正确。

------

这是：

非常重要的区别。

------

# 第十三节：Flow Engine 实战

我们的流程系统：

不要：

全部：

```
int
```

例如：

节点：

```
type NodeID int

type FlowID int

type UserID int
```

------

状态：

```
type NodeStatus int
```

------

时间：

```
type TimeoutSeconds int
```

------

这样：

系统：

更安全。

------

# 第十四节：企业类型设计原则

记住：

## 原则 1

能表达业务：

就不要只用基础类型。

------

差：

```
func Create(int,string,int)
```

------

好：

```
func Create(UserID,Name,Age)
```

------

## 原则 2

不要为了炫技创建大量类型。

------

类型：

应该：

提高理解。

不是：

制造复杂。

------
